# Project memory layer

This directory is the **canonical, tracked** memory of `APIskillBuilderSkill`. Everything here is part of the repo and visible to collaborators and any AI agent working on the project.

There are two other layers, kept separate from this one on purpose:

| Layer | Location | Tracked? | What lives there |
| --- | --- | --- | --- |
| **In-repo canonical** (you are here) | `docs/` | yes | ADRs and session-by-session progress notes |
| **Live structure graph** | `graphify-out/` (repo root) | no — auto-regenerated by graphify | `GRAPH_REPORT.md`, `graph.json`. Rebuilt by the post-commit hook |
| **Operator's centralized vault** | `~/.claude/vault/` | no — lives outside the repo | Cross-project graphify snapshots at `graphify/APIskillBuilderSkill/`; reusable concept notes at `zettel/concepts/` |

The split rule: anything that should ship with the repo or guide a contributor goes in `docs/`. Anything that's a working surface for the operator's personal Obsidian / graphify workflow goes in `~/.claude/vault/`.

## Directory layout

```
docs/
├── README.md                         (this file)
├── settings.local.json.template      Harness deny-rules template for AI sessions
├── progress/                         Session-by-session notes
│   ├── TEMPLATE.md
│   └── YYYY-MM-DD-<slug>.md
└── decisions/                        ADRs, numbered monotonically
    ├── TEMPLATE.md
    └── NNNN-<slug>.md
```

## Slash commands

This repo ships project-level overrides of `/resume` and `/save` (in `.claude/commands/`) that know about this exact layout:

- **`/resume`** reads `docs/progress/` (most recent), `graphify-out/GRAPH_REPORT.md`, three most recent `docs/decisions/`, and summarizes where we left off.
- **`/save`** writes a new `docs/progress/YYYY-MM-DD-<slug>.md`, optionally creates an ADR for material design decisions, and routes reusable cross-project concepts to `~/.claude/vault/zettel/concepts/`.

## How to add an entry yourself

- **New progress note**: copy `progress/TEMPLATE.md` → `progress/YYYY-MM-DD-<short-slug>.md` and fill it in.
- **New ADR**: read the highest-numbered file in `decisions/`, use the next number, copy `decisions/TEMPLATE.md` → `decisions/NNNN-<short-slug>.md`. Wikilink to related ADRs with `[[NNNN-other-decision]]`.
- **New concept note** (cross-project): create at `~/.claude/vault/zettel/concepts/<concept-slug>.md`. Don't put concept notes in this repo's `docs/` — they're meant to outlive any single project.

## Obsidian + MCP

If you use Obsidian, open `~/.claude/vault/` as your single working vault across projects. Configure the Obsidian MCP server (e.g. [`iansinnott/obsidian-claude-code-mcp`](https://github.com/iansinnott/obsidian-claude-code-mcp)) to point at the same vault. Then `/resume` and `/save` interact with it via MCP rather than raw filesystem.

For contributors who don't use Obsidian, this `docs/` directory is sufficient — open it as a vault directly, or read it as plain markdown.

## graphify

This repo has a graphify knowledge graph at `graphify-out/` (gitignored). On a fresh clone run `graphify update .` once to populate it; the post-commit hook keeps it fresh thereafter. Install the hooks once with `graphify hook install` and `graphify claude install` (the latter writes `.claude/settings.json`, which is tracked).
